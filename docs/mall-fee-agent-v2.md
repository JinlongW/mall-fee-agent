## 一、产品定位：不是另一个 ERP，而是让现有系统变聪明

### 1.1 客户现状与痛点

```
客户已有的系统：
├── ERP/财务系统（用友/金蝶/SAP）→ 记账、开票
├── 物业管理系统（泓远/快鲸/自建）→ 合同管理、基础费用
├── POS/收银系统 → 商户营业数据
├── 抄表系统（IoT/手动）→ 水电读数
└── OA 系统 → 审批流程

现有系统的"笨"在哪里：
├── 合同条款变更 → 需手动修改费用规则，经常忘记更新
├── 复杂租金模式 → 只能配固定规则，"两者取高"要人工判断
├── 阶梯计算 → 很多系统不支持阶梯扣率
├── 免租期+递增+保底组合 → 规则叠加后系统算不对
├── 异常检测 → 系统不会主动告诉你"这个月费用不对"
├── 争议处理 → 商户问"为什么这么多"，系统给不出解释
└── 催缴 → 系统只生成账单，不会智能催缴
```

### 1.2 我们的产品 = 现有系统的 AI 外挂

```
┌──────────────────────────────────────────────┐
│     客户现有的物业/ERP 系统（不动它）          │
└───────────────────┬──────────────────────────┘
                    │
        ┌───────────▼───────────┐
        │  🧠 费用核算 AI Agent │  ← 我们的产品
        │                       │
        │  读取合同 → 理解规则   │
        │  读取数据 → 智能核算   │
        │  生成账单 → 推送通知   │
        │  对账催缴 → 自动闭环   │
        │  异常发现 → 主动预警   │
        └───────────┬───────────┘
                    │
        ┌───────────┴───────────┐
        │  输出方式（三选一）：   │
        │  • 推送回现有系统      │
        │  • 独立生成 PDF 账单   │
        │  • 生成 Excel 导入表   │
        └───────────────────────┘
```

**一句话**：不替代你的系统，让你的系统从"手动配置规则"变成"AI 自动理解合同"。

## 二、业务逻辑深度拆解（核心竞争力）

### 2.1 购物中心费用全景

```
商户每月应缴 = 租金 + 物业费 + 水电费 + 其他费用

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

一、租金（最复杂，占 Agent 价值的 60%）

  模式 A：固定租金
  ├── 月租金 = 面积 × 单价（元/㎡/月）
  ├── 递增：每年/每N年递增 X%
  │   ├── 递增基准 = 签约时单价 × (1+递增率)^年数
  │   └── 递增基准 = 上月租金 × (1+递增率)
  ├── 免租期处理（3 种方式）：
  │   ├── 直接减免：1-3月租金=0
  │   ├── 均摊：总租金不变，每月少一点
  │   └── 先交后返：正常交，到期返还
  └── ⚠️ 边界：免租期内物业费是否也免？看合同！

  模式 B：扣率租金
  ├── 月租金 = 月营业额 × 扣率%
  ├── 业态差异：
  │   ├── 餐饮 12-18%（堂食/外卖可能不同）
  │   ├── 零售 8-15%
  │   ├── 娱乐 15-25%
  │   └── 超市 5-8%（大面积低扣率）
  ├── 保底：MAX(扣率, 保底) → 本质就是"两者取高"
  └── 营业额来源：POS / 商户自报 / 税务数据

  模式 C：两者取高（最常见！）
  ├── 月租金 = MAX(固定租金, 营业额 × 扣率)
  ├── 每月独立比较
  └── ⚠️ 边界：保底租金是否随递增一起递增？

  模式 D：阶梯扣率
  ├── 营业额 ≤ 10万部分：扣率 8%
  ├── 10-30万部分：扣率 12%
  ├── > 30万部分：扣率 15%
  └── 分段计算，累加合计

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

二、物业费（较简单，但有细节）
  ├── 月物业费 = 面积 × 物业单价
  ├── 公摊：公共区域水电/空调/电梯 按面积分摊
  ├── 包含/不包含：有些合同物业费包含空调费，有些另算
  └── ⚠️ 边界：免租期物业费是否减免？

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

三、水电费（数据驱动，需对接抄表）
  电费：
  ├── 用量 = 本期读数 - 上期读数
  ├── 分时计价：峰时/谷时/平时 不同单价
  ├── 阶梯电价：用量越高单价越贵
  ├── 公摊：公共照明/电梯/空调 按面积分摊
  ├── 损耗：变压器损耗 1-3% 加到商户头上
  └── ⚠️ 边界：换表/故障读数的处理

  水费：
  ├── 用量 = 本期读数 - 上期读数
  ├── 污水处理费：用水量 × 附加费率
  ├── 公摊：公共绿化/清洁用水
  └── ⚠️ 边界：总表 vs 分表 差异如何处理

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

四、其他费用
  ├── 推广费/广告费：固定 or 按营业额比例
  ├── POS系统使用费：固定月费
  ├── 垃圾清运费：固定 or 按面积
  ├── 仓储费：按面积/月
  ├── 装修保证金：入住时收，退租时退还
  ├── 进场费：一次性
  └── 违约金/滞纳金：逾期按日 X%
```

### 2.2 边界条件清单（Agent 最容易出错的地方）

```
以下是你作为专家需要帮 Agent 定义的边界规则：

1. 免租期边界：
   ├── 免租期内租金=0，物业费是否也免？
   ├── 免租期跨月怎么算？（如免租期 1.15-3.15）
   └── 递增从免租期后开始还是从签约日开始？

2. 递增边界：
   ├── 递增是按"签约日周年"还是"自然年"？
   ├── 递增时，保底租金是否一起递增？
   └── 递增基数是"原单价"还是"上期单价"？

3. 营业额边界：
   ├── 商户不报营业额怎么办？（用保底 or 用同业态平均？）
   ├── 营业额数据延迟（如 6 月才能拿到 5 月数据）
   └── 退货/退款是否从营业额中扣除？

4. 抄表边界：
   ├── 换表时新旧表读数如何衔接？
   ├── 抄表失败/异常读数如何处理？
   └── 公摊电费的分母是所有商户还是仅运营中的？

5. 合同变更边界：
   ├── 合同中途变更面积怎么算？（按天折算？）
   ├── 商户中途退租，剩余月份费用如何处理？
   └── 合同续签但条款变更，新旧规则如何切换？

6. 多铺位边界：
   ├── 一个商户多个铺位，是否合并计算？
   └── 不同铺位不同合同条款，管理费如何分摊？
```

### 3.3 费用核算引擎核心逻辑

```python
class MallFeeEngine:
    def calc_monthly(self, rules, period, meters, sales):
        bill = Bill(merchant=rules.merchant, period=period)

        # 1. 租金
        if self._in_free_rent(rules, period):
            bill.rent = 0
        else:
            fixed = self._calc_fixed_rent(rules, period)
            turnover = max(sales * rules.rent.turnover.rate, rules.rent.turnover.minimum)
            if rules.rent.type == "take_higher":
                bill.rent = max(fixed, turnover)
            elif rules.rent.type == "tiered":
                bill.rent = self._calc_tiered(rules.rent.tiers, sales)
            # ... 其他模式

        # 2. 物业费
        bill.property_fee = rules.unit.area * rules.property_fee.price

        # 3. 电费（阶梯+分时+损耗+公摊）
        elec_usage = meters.elec_current - meters.elec_previous
        bill.electricity = self._calc_electricity(elec_usage, rules.utilities.electricity)

        # 4. 水费 + 污水处理
        water_usage = meters.water_current - meters.water_previous
        bill.water = water_usage * rules.utilities.water.price
        bill.sewage = water_usage * rules.utilities.water.sewage_rate

        # 5. 其他 + 汇总 + 异常检测
        bill.other = sum(self._calc_other(rules.other_fees, period).values())
        bill.total = bill.rent + bill.property_fee + bill.electricity + bill.water + bill.sewage + bill.other
        bill.anomalies = self._detect_anomalies(bill, merchant.history[-6:])
        return bill
```

### 3.4 合同解析 Agent — Prompt 设计

```
CONTRACT_PARSE_PROMPT:

你是购物中心租赁合同费用规则提取专家。

从合同文本中提取费用计算规则，输出 JSON。需要提取：
1. 商户信息（名称、业态）
2. 铺位信息（编号、面积、楼层）
3. 租赁期限 + 免租期（起止日期、是否包含物业费）
4. 租金模式（固定/扣率/两者取高/阶梯）
5. 固定租金（单价、递增规则：频率/比率/基准）
6. 扣率租金（扣率%、保底金额、保底是否递增）
7. 物业费（单价、是否含空调、公摊比例）
8. 水电费（电价类型/阶梯/分时、水价、污水处理费）
9. 其他费用（推广费、POS费、垃圾费等）
10. 滞纳金（费率、宽限天数）

特别注意：
- 递增规则要区分"按签约价递增"还是"按上期递增"
- 免租期要区分"租金免"还是"全免（含物业费）"
- 保底租金是否随递增一起变化
- 阶梯扣率断点是"含"还是"不含"
```
